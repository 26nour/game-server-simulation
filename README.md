## 🌟 نظرة عامة
**Game Server Pro** هو تطبيق محاكاة متكامل لخادم ألعاب يعمل في بيئة متعددة الخيوط، مصمم لتوضيح وإدارة:
- **إدارة اللاعبين** (انضمام، مغادرة، سلوكيات مختلفة).
- **إنشاء وإدارة غرف الألعاب** (كل غرفة تتسع لأربعة لاعبين).
- **تخصيص الموارد** (CPU، الذاكرة، النطاق الترددي، اتصالات قاعدة البيانات، نسخ الألعاب) باستخدام **سيمافورات** و **أقفال** لضمان التزامن الآمن.
- **جدولة الطلبات** بأولوية (0 عالية → 10 منخفضة).
- **كشف الجمود وحله** باستخدام خوارزمية DFS على رسم بياني تخصيص الموارد.
- **محاكاة حالات التضارب** (مع وبدون أقفال) لتوضيح أهمية المزامنة.
- **واجهة رسومية احترافية** تحتوي على لوحات تحكم، رسوم بيانية حية، بث مباشر للعبة، وسجلات مفصلة.

المشروع مبني على مكتبة `customtkinter` لواجهة حديثة و `matplotlib` للرسوم البيانية، ويعمل بنظام الأحداث غير المتزامن لضمان استجابة سلسة.

---

## 🎯 الأهداف التعليمية
- فهم آلية عمل الخوادم المتعددة الخيوط.
- تطبيق مفاهيم التزامن (Mutex، Semaphore) في بيئة حقيقية.
- تصميم نظام جدولة بأولوية.
- تنفيذ خوارزمية كشف الجمود (DFS على Resource Allocation Graph).
- رصد وتحليل أداء النظام (زمن الاستجابة، الإنتاجية).
- محاكاة حالات التضارب وتأثيرها على صحة البيانات.
- بناء واجهة مستخدم تفاعلية تعكس حالة النظام لحظياً.

### المتطلبات الأساسية
- Python 3.8 أو أحدث.
- المكتبات المذكورة في `requirements.txt`:

```txt
customtkinter>=5.2.0

```

### خطوات التثبيت
1. استنساخ المشروع أو تنزيل الملفات.
2. تثبيت المكتبات المطلوبة:
   ```bash
   pip install -r requirements.txt
   ```
3. تشغيل التطبيق:
   ```bash
   python game_gui.py
   ```

> **ملاحظة:** سيتم فتح شاشة تسجيل الدخول. يمكن استخدام الحسابات المذكورة في الشاشة (admin / operator1 / monitor1 إلخ) مع كلمات المرور المعروضة.

## 🧩 المكونات الأساسية

### النواة الأساسية للخادم (`game_server_core.py`)
- **`GameResource`**: يمثل مورداً مع سيمافور للحد من الوصول المتزامن، وقفل لتحديث الحالة وسجل تاريخي.
- **`GameSession`**: غرفة لعبة تتسع لأربعة لاعبين، تحتفظ بوقت البدء والموارد المخصصة.
- **`DeadlockDetector`**: يحتفظ بجداول التخصيص والطلب، ويطبق DFS لكشف الدورات (الجمود).
- **`PriorityScheduler`**: جدولة طلبات المطابقة بأولوية (0-10).
- **`PerformanceMonitor`**: يسجل أوقات الاستجابة والإنتاجية.
- **`GameServerCore`**: الكيان الرئيسي:
  - يحتوي على موارد (5 أنواع) مع سيمافورات.
  - إدارة اللاعبين والألعاب باستخدام أقفال (`RLock`).
  - معالجة طلبات المطابقة عبر خيوط مستهلكة (`consumer threads`).
  - مراقبة دورية للجمود.
  - دالة `simulate_batch_join(count, use_lock)` لمحاكاة إضافة لاعبين مع أو بدون قفل، وإرجاع تفاصيل عن اللاعبين المطرودين والناجين.

### محاكاة العملاء (`game_client_sim.py`)
- **`GameClient`**: يمثل لاعباً بسلوك معين، يعمل في خيط مستقل.
  - سلوكيات: `normal`, `aggressive`, `casual`, `afk`.
  - يحدد زمن الانتظار بين الإجراءات واحتمالية تنفيذ إجراء (مغادرة أو طلب مطابقة).
- **`ClientManager`**: يدير مجموعة من العملاء، يضيف/يزيل عملاء، ويتحكم في دورة حياتهم.

### الواجهة الرسومية (`game_gui.py`)
- **`ToastNotification`**: نظام إشعارات منبثقة (تختفي تلقائياً) بأنواع (info, success, warning, error).
- **`FloatingButton`**: زر عائم بتأثير نبض، مع تلميح عند المرور.
- **`LiveGameCanvas`** (النسخة الأصلية) و **`LiveGameCanvasEnhanced`** (المستوردة من `live_game_enhanced.py`): عرض حي للعبة مع حركة اللاعبين والرصاص.
- **`LoginScreen`**: شاشة تسجيل الدخول مع قائمة حسابات جاهزة وزر سريع لملء البيانات.
- **`GameServerGUI`**: النافذة الرئيسية، تحتوي على:
  - شريط عنوان (معلومات المستخدم، تبديل المظهر، خروج).
  - 10 تبويبات رئيسية.
  - شريط حالة (حالة الخادم، الساعة، مؤشر نبض).
  - 5 أزرار عائمة (إضافة لاعب، بدء/إيقاف الخادم، إعادة ضبط، فتح البث المباشر، تشغيل المحاكاة التلقائية للتضارب).
  - تحديثات دورية لجميع العناصر (كل ثانية).

### البث المباشر المحسن (`live_game_enhanced.py`)
- يوفر حركة لاعبين طبيعية (تغيير اتجاه عشوائي، سرعات مختلفة).
- إطلاق رصاص متنوع الألوان نحو أقرب لاعبين في نفس الغرفة.
- تأثيرات إصابة (وميض) وتأثيرات موت (انفجار متدرج).
- عند موت لاعب، تتم إزالته من السيرفر بعد 0.5 ثانية (محاكاة للخروج القسري).
- يتم مزامنة اللاعبين مع السيرفر الحقيقي باستمرار.

## 📑 شرح التبويبات والوظائف

### 🎛️ Control (التحكم)
- **Server Controls**: أزرار لبدء الخادم، إيقافه، وإعادة ضبطه.
- **Add Players**: شريط تمرير لتحديد عدد اللاعبين، قائمة منسدلة لاختيار السلوك، وزر لإضافتهم.
- **Race Condition Simulation**: مفتاح لتفعيل استخدام القفل/السيمافور، وزران لتشغيل محاكاة التضارب (بدون قفل / مع قفل) مع عرض النتائج في السجلات.
- **Quick Stats**: عرض عدد اللاعبين، الألعاب، الطلبات المعلقة، والإنتاجية.
- **Resource Overview**: أشرطة تقدم مضغوطة لكل مورد.

### 📊 Dashboard (لوحة المعلومات)
- أربع بطاقات KPI: عدد اللاعبين، الألعاب النشطة، نسبة نجاح المطابقة، الإنتاجية.
- رسم بياني رئيسي يوضح تطور عدد اللاعبين والألعاب عبر الزمن (آخر 60 نقطة).

### 👥 Players (اللاعبون)
- جدول يعرض جميع اللاعبين: المعرف، الاسم، السلوك، الحالة، معرف اللعبة، الصحة (من البث المباشر)، وزر طرد لكل لاعب (للمستخدمين المخولين).
- زر **Kick All** لطرد جميع اللاعبين (للمشرفين).

### 🎮 Games (الألعاب)
- جدول يعرض الغرف النشطة: معرف اللعبة، الاسم، عدد اللاعبين، السعة، المدة، الحالة.
- زر **End** لإنهاء لعبة معينة (للمشرفين).

### ⚙️ Resources (الموارد)
- بطاقات تفصيلية لكل مورد تعرض الاستخدام الحالي كنسبة مئوية وشريط تقدم.
- رسم بياني يوضح تاريخ استخدام الموارد خلال آخر 40 نقطة زمنية.

### 📋 Scheduling (الجدولة)
- عرض قوائم الانتظار ذات الأولوية (من 0 إلى 10)، مع شريط تقدم يوضح حجم الطلبات في كل أولوية.

### 🔒 Deadlock (الجمود)
- **أزرار**:
  - **Detect Deadlock**: تشغيل خوارزمية DFS لكشف الجمود وعرض النتيجة في مربع النص.
  - **Simulate Deadlock**: إنشاء سيناريو جمود كلاسيكي (دائري) بين عمليتين.
  - **Resolve & Clear**: مسح جداول التخصيص والطلب وحل الجمود.
- مربع نص يعرض تفاصيل الجمود أو حالة الأمان.

### 📈 Statistics (الإحصائيات)
- مربع نص يعرض إحصائيات الأداء (زمن الاستجابة، الإنتاجية، عدد الطلبات) وإحصائيات المطابقة (عدد اللاعبين المنضمين/المغادرين، الألعاب المنشأة/المنتهية، محاولات المطابقة، النجاح، نسبة النجاح).
- رسمان بيانيان: **Response Times** (زمن الاستجابة بالمللي ثانية) و **Throughput** (الإنتاجية).

### 🎬 Live View (البث المباشر)
- يعرض ساحة معركة حية مع اللاعبين الحقيقيين الموجودين على الخادم.
- يتحرك اللاعبون تلقائياً، يطلقون النار على بعضهم، وتظهر تأثيرات الإصابة والموت.
- عند موت لاعب، يغادر السيرفر فعلياً.
- يوجد شريط أحداث يوضح آخر الأحداث (انضمام، إصابة، موت، مغادرة).

### 📝 Logs (السجلات)
- نافذة سجل نصي تعرض جميع الأحداث والرسائل الصادرة عن النظام.
- أزرار **Clear** لمسح السجل و **Export** لحفظ السجل في ملف نصي.

## 🧭 الأزرار العائمة (FAB)
- **➕ Add Player**: يضيف لاعباً واحداً بسلوك عشوائي (يحتاج صلاحية).
- **▶ / ⏹ Start/Stop Server**: تشغيل/إيقاف الخادم.
- **↺ Reset Server**: إعادة ضبط الخادم (يوقف، يمسح البيانات، ويعيد التشغيل).
- **🎬 Live View**: ينتقل مباشرة إلى تبويب البث المباشر.
- **⚡ Auto Race**: يشغل/يوقف المحاكاة التلقائية للتضارب (تضارب عشوائي مع أو بدون قفل كل 5-10 ثوانٍ). متاح للمشرفين فقط.

## 🔐 نظام تسجيل الدخول والأدوار
- يتضمن النظام قاعدة بيانات محاكاة للمستخدمين (قاموس `USERS_DB`).
- الأدوار:
  - **Admin**: صلاحيات كاملة (جميع العمليات).
  - **Operator**: صلاحيات محدودة (عرض، إضافة لاعب، طرد لاعب).
  - **Monitor**: صلاحيات عرض فقط.
- الألوان: `#ff6b35` للمشرف، `#4caf50` للمشغل، `#2196f3` للمراقب.
- شاشة الدخول تعرض قائمة الحسابات مع كلمات المرور وزر **Use** لتعبئة البيانات بسرعة.

## 🏃 محاكاة التضارب (Race Condition)
- توفر النواة دالة `simulate_batch_join(count, use_lock)`:
  - **مع القفل (use_lock=True)**: تضاف اللاعبين بشكل متسلسل مع قفل يضمن عدم تداخل عمليات الإضافة، مما يؤدي إلى نتائج صحيحة (جميع اللاعبين ينضمون).
  - **بدون قفل (use_lock=False)**: تضاف اللاعبين عبر خيوط متعددة دون مزامنة، مما يؤدي إلى فقدان بعض اللاعبين (بسبب سباق على البيانات) وظهور اختلاف بين العدد المتوقع والفعلي.
- في الواجهة، يمكن تشغيل محاكاة التضارب يدوياً (زر **Without Lock** / **With Lock**) أو تلقائياً عبر زر **Auto Race** الذي ينفذ محاكاة عشوائية بشكل دوري.
- تسجل النتائج في السجلات (عدد اللاعبين المفقودين، الألعاب المنشأة، قوائم اللاعبين المطرودين والناجين).

## ✨ الميزات المتقدمة
- **تحديثات لحظية**: جميع البيانات تُحدث بشكل دوري (كل ثانية) دون الحاجة لإعادة تحميل الصفحة.
- **إشعارات أنيقة**: تظهر في أعلى اليمين وتختفي تلقائياً.
- **وضعان للمظهر**: داكن وفاتح (قابل للتبديل بزر في شريط العنوان).
- **نظام صلاحيات**: يتحكم في الوصول إلى العمليات الحساسة.
- **البث المباشر**: تفاعلي بالكامل، مع تأثيرات بصرية وإزالة تلقائية للاعبين الموتى.
- **سجلات مفصلة**: تسجل كل حدث (انضمام، مغادرة، إصابة، موت، إنشاء لعبة، إلخ).
- **تصدير السجلات**: حفظ السجلات إلى ملف نصي.
- **كشف الجمود**: خوارزمية DFS مع إمكانية محاكاة وحل.
